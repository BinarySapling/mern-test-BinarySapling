# Deployment Guide

## Backend (Render)
1. Push your code to GitHub.
2. Go to Render.com and create a new Web Service.
3. Connect your GitHub repository.
4. Set the root directory to `backend`.
5. Set the build command to `npm install`.
6. Set the start command to `npm run start:prod`.
7. Add environment variables from `backend/.env.example`:
   - PORT
   - MONGO_URI
   - JWT_SECRET
   - FRONTEND_URL
8. Optionally, use `render.yaml` for auto-deploy configuration.
9. Deploy the service.

## Frontend (Vercel)
1. Push your code to GitHub.
2. Go to Vercel.com and import your project from GitHub.
3. Set the root directory to `frontend`.
4. Add environment variable from `frontend/.env.example`:
   - VITE_API_URL (should be your deployed backend API URL)
5. Deploy the project.

## Notes
- Ensure CORS is configured on the backend for your frontend URL.
- Use secure, production-ready values for all secrets.
- For local development, copy `.env.example` to `.env` and fill in your values.
- After deployment, test all endpoints and UI flows.
